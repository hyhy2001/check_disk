# Check Disk 💾

> A high-performance disk usage monitoring CLI for Linux servers, powered by a native Rust core with a pure-Python fallback.

`check_disk` is designed to rapidly scan heavily-loaded file systems (like NFS/SAN or dense local storage). It bypasses Python's Global Interpreter Lock (GIL) via a precompiled Rust module (`fast_scanner.abi3.so`) for true multi-threaded I/O, seamlessly falling back to a pure-Python engine if the binary is unavailable.

---

## 🚀 Quick Start

Get up and running in less than 5 minutes.

### 1. Initialization
Create the default configuration (`disk_checker_config.json`):
```bash
python disk_checker.py --init --dir /data/shared
```

### 2. Setup Teams & Users
```bash
# Define teams
python disk_checker.py --add-team backend
python disk_checker.py --add-team frontend

# Assign users to teams
python disk_checker.py --add-user alice bob carol --team backend
python disk_checker.py --add-user dave eve --team frontend
```

### 3. Run & View Reports
```bash
# Execute the scan
python disk_checker.py --run

# View the generated report summary
python disk_checker.py --show-report --files disk_usage_report.json

# View detailed per-user metrics (files & directories)
python disk_checker.py --check-users alice bob
```

## ✨ Features

- **Blazing Fast Scanning (Native Rust)**
  - Parallel `WalkBuilder` traversal for zero-GIL, multi-threaded I/O.
  - Auto-scaling worker threads automatically optimized for high-latency systems.
  - Minimal syscall overhead (single `stat()` per entry).
  - Smart skipping: handles hard-links, snapshots, sparse files natively, and automatically ignores system mounts (`proc`, `sys`, `dev`).
- **Reliable Pure-Python Fallback**
  - Uses `os.scandir()` with work-stealing for pure Python environments.
- **Advanced Report Generation**
  - **K-way Merge in Rust**: Assembles massive file reports streaming to disk efficiently (`O(chunks)` RAM).
  - Generates detailed JSONs for top directories and largest files per user, safe for 75M+ file trees.
- **User & Team Quota Tracking**
  - Map filesystem UIDs to custom Teams and Users for granular insights.
  - Track accessibility boundaries with out-of-the-box permission error reporting.

## ⚙️ Configuration Format

The tool reads from `disk_checker_config.json`, which is auto-generated by the `--init` command.

```json
{
  "directory": "/data/shared",
  "output_file": "disk_usage_report.json",
  "use_rust": true,
  "teams": [
    { "name": "backend",  "team_id": 1 },
    { "name": "frontend", "team_id": 2 }
  ],
  "users": [
    { "name": "alice", "team_id": 1 },
    { "name": "bob",   "team_id": 1 }
  ]
}
```

## 💻 CLI Reference

### Environment Setup
| Command | Description |
|---------|-------------|
| `--init --dir <path>` | Initializes config file with root directory. |
| `--add-team <name>` | Adds a new team. |
| `--add-user <u1> <u2> --team <t>` | Associates users with a specific team. |
| `--remove-user <name>` | Removes a user from the tracking config. |
| `--list` | Lists current teams and users. |

### Scanning Options
| Command | Description |
|---------|-------------|
| `--run` | Runs a standard scan over configured directory. |
| `--run --user <u1> <u2>` | Targeted scan mode. Incredibly fast for tracking specific users only. |
| `--run --workers <N>` | Manually set parallel workers (Applicable for *Python fallback* only). |
| `--run --date --prefix <name>` | Appends timestamp and prefix to output file names. |

### Inspecting & Exporting Reports
| Command | Description |
|---------|-------------|
| `--show-report --files <f1.json>` | Shows summary output in a clean terminal table format. |
| `--show-report --files <f1> <f2> --compare-by growth` | Differencing engine between temporal reports. |
| `--check-users <u1> <u2>` | Displays largest files and directories for specific users. |
| *(Script)* `python scripts/export_user_reports.py ...` | Exports JSON detailed user reports to plain text formatting. |

---

## 📊 Output Formats

All results are exported as standard JSON streams:

- `disk_usage_report.json`: Overall summary, global system stats, team usage, and UID mappings.
- `detail_report_dir_{user}.json`: Largest directories owned by specific users.
- `detail_report_file_{user}.json`: Largest files globally owned by a user, sorted descending by size.
- `permission_issues.json`: Records specific paths that couldn't be scanned due to permissions.

## 🏗 Architecture & Modules

### `fast_scanner.abi3.so` (Rust Native Extension)
Precompiled PyO3 module matching Python's `abi3` specification.
- **`fast_scanner.scan_disk()`**: Parallel I/O walker. Calculates blocks accurately and aggregates sizes per UID.
- **`fast_scanner.merge_write_user_report()`**: K-way `BinaryHeap` merge implementation converting thousands of Temporary TSV chunks to sorted JSON files natively.

### Source Code Mapping
- `disk_checker.py`: Main CLI entrypoint.
- `src/disk_scanner.py`: Proxy layer, switching gracefully between Rust's optimized scanner and legacy work-stealing Python fallback depending on availability.
- `src/report_generator.py`: Controller transforming raw `ScanResult` instances to specific JSON file payloads.
- `scripts/`: Assorted data lifecycle automations like backfilling metric ID's and exporting plain text datasets.

## ⚡ Performance & Memory Notes

**Engine Throughput Guidelines:**
| Scenario | Engine | Expected Time |
|----------|--------|---------------|
| NVMe local, 10M files | Rust | < 5 min |
| SSD local, 50M files | Rust | 5–15 min |
| NFS/SAN high-latency, 75M files | Rust (Auto-scaled) | 15–40 min |
| HDD single disk | Rust (I/O Bound) | 30–90 min |
| Legacy scan, 75M files | Python Fallback | 60–180 min |

**Memory Efficiency (Rust vs Legacy Python)**
Switching to Rust drops peak memory RAM footprint from `~5 GB` (due to heavy Python datastructures) down to **`~1.5 GB`** for massive `75+ million` inode traversals by aggressively managing garbage collection and buffering stream data directly bounded to disks.
