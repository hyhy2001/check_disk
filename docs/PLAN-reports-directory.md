# 📂 Project Plan: Reports Directory Output

## 1. Overview
The goal is to refactor the report generation mechanism so that all JSON reports generated by the disk scanner are cleanly organized within a root `reports/` directory. This ensures a scalable, predictable file structure instead of scattering output files.

## 2. Project Type
**BACKEND** (CLI / File System Management)

## 3. Success Criteria
- [ ] A central `reports/` directory is utilized as the default root for all generated report outputs.
- [ ] The main usage report is correctly formatted as `reports/{prefix}_disk_usage_report_<yyyymmdd>.json`.
- [ ] The permission issues report acts as a sibling and strictly formatted as `reports/{prefix}_permission_issues.json` (Without the date suffix).
- [ ] Detail user reports are reliably grouped in `reports/detail_users/` ensuring their specific prefix definitions and avoiding date suffixes.
- [ ] Both reading mechanisms (like `--check-users`) and writing resolve perfectly to this folder map.

## 4. Tech Stack
- **Python 3**: Utilizing `os.path` and `os.makedirs` for cross-platform, robust directory creation and path joining.

## 5. File Structure
Target generated file structure:
```text
reports/
 ├── {prefix}_disk_usage_report_<yyyymmdd>.json
 ├── {prefix}_permission_issues.json
 └── detail_users/
      ├── {prefix}_detail_report_dir_{user}.json
      └── {prefix}_detail_report_file_{user}.json
```
